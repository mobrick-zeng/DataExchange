## 一、一句話總覽

跨銀行「前置調解債權申報／還款管理」平台，為**前後端整合的全棧系統**：

> 最大債權行（主辦）建立案件並**代填**各債權行明細 → 邀請其他債權行 → 發布 → 其他債權行**確認／回報異議** → 全數確認後進入**還款中**、逐月登記還款 → **結清**或**毀諾終止**。

帳號採**管理員邀請制**，資料存於 **PostgreSQL**，權限一律由**後端強制（Server-side RBAC）**。前端已**無任何 localStorage 資料層**，全部改呼叫後端 REST API。

## 二、技術架構

| 層 | 技術 | 位置 |
|---|---|---|
| 前端 | React 18 + TypeScript + Vite 5 + Tailwind 3 + React Router 6 + Zustand | `src/` |
| 前端資料存取 | 統一經 `src/services/api.ts`（`apiFetch`，自動附 JWT）呼叫後端 | `src/services/` |
| 後端 | Fastify 4 + Prisma 5 + Zod（`@fastify/jwt`、bcryptjs、`@fastify/rate-limit`、`@fastify/cors`） | `backend/src/` |
| 資料庫 | PostgreSQL 16 | Docker |
| 部署 | 本機開發（Vite + tsx）或全棧 Docker（db + api + web + Caddy 反向代理） | `docker-compose.fullstack.yml` |

### 認證與帳號模型（邀請制）
- **登入**：銀行代碼 + Email + 密碼三層驗證 → 回傳 JWT（效期 **8 小時**，payload 含 `userId / role / bankCode / name`）。失敗訊息一律通用「銀行／帳號／密碼不正確」，避免帳號列舉。
- **登入鎖定**：在「帳號＋銀行正確、僅密碼錯」時累計失敗，連續 **5 次** → 鎖定 **15 分鐘**；管理員可手動解鎖。
- **建立帳號**：管理員發一次性邀請碼（效期 **3 天**、免寄信，由管理員手動遞送）→ 受邀者於 `/register?code=…` 設定姓名與密碼完成註冊，帳號即 **`ACTIVE`**。
- **密碼重設**：使用者於「忘記密碼」送出申請（一律通用回應）→ 管理員核發一次性重置碼 → 使用者於 `/register?code=…`（用途為重置）僅設定新密碼。管理員亦可主動替帳號核發重置碼。
- **已移除（舊機制）**：公開自助註冊、Email 6 碼 OTP 驗證頁、待審核頁、舊版重設密碼頁、Demo 信箱抽屜。

### 案件狀態機（後端實際使用）
```
DRAFT → PENDING_CONFIRMATION → IN_REPAYMENT → SETTLED / TERMINATED
```
| 狀態 | 意義 | 可執行動作（限主辦） |
|---|---|---|
| `DRAFT` | 建立中，主辦代填、邀請其他債權行 | 編輯明細、邀請 CO_BANK、發布 |
| `PENDING_CONFIRMATION` | 已發布，待各其他債權行確認 | 修正被回報異議（DISPUTED）的資料 |
| `IN_REPAYMENT` | 全數確認、方案生效，逐月更新（停留最久） | 登記還款、結清、毀諾終止 |
| `SETTLED` | 已結清 | — |
| `TERMINATED` | 毀諾／終止 | — |

其他債權行對案件的確認流程：`NOT_REQUIRED`（主辦本身）／`PENDING`／`CONFIRMED`／`DISPUTED`。

### 角色（RBAC）
- `ADMIN` 平台管理員：帳號審核、邀請、密碼重置、稽核、全案可見。
- `BANK_STAFF` 銀行人員：**角色因案動態認定** —— 建立案件者所屬銀行即該案「最大債權行（主辦，MAIN）」，可編輯；受邀者為「其他債權行（CO_BANK）」，僅檢視＋確認／異議。
- `PLATFORM_AUDITOR` 平台稽核：全案唯讀＋稽核，並為唯一可見 `internalTotal`（內部總額）者（方案 B）。
- `VIEWER`：定義存在，目前流程未實際使用。

## 三、如何執行

需求環境：**Node.js 20+**、**Docker Desktop**。

### 方式一：本機開發（開發最快）

```bash
# 專案根目錄

# 1) 啟動資料庫（Docker，Postgres 對外埠 5544）
cd backend
docker compose up -d

# 2) 安裝相依、套用 migration 與種子（首次或重置資料時）
npm install
npx prisma migrate deploy
npm run prisma:seed          # SEED_MODE 預設 demo，建立 5 組示範帳號

# 3) 啟動後端 API（http://localhost:4000，tsx watch 熱重載）
npm run dev

# 4) 另開終端，回專案根目錄啟動前端
cd ..
npm install
npm run dev                  # http://localhost:5173（Vite HMR）
```

瀏覽器開 **http://localhost:5173**，以第七節的 Demo 帳號登入。前端 API 位址由 `VITE_API_BASE` 決定，未設定時預設 `http://localhost:4000`。

### 方式二：全棧 Docker（驗收／部署）

一份原始碼、兩種跑法。dev 與 fullstack **讀同一份原始碼**，差別只在「怎麼跑」；`--build` 就是 dev → fullstack 的唯一「合併」動作，會以目前原始碼重新編譯映像。

正式部署前，先依情境擇一複製環境變數範本為 `.env`（兩份範本檔內皆有詳細註解）：

```bash
# OCI／雲主機（只有公網 IP、走 HTTP）
cp .env.production.oci.example .env

# 或：有正式網域（啟用 HTTPS）
cp .env.production.domain.example .env
```

```bash
# Demo 版（含 5 組示範帳號）
docker compose -f docker-compose.fullstack.yml -p mediation-demo up -d --build

# 正式版（只建 admin + 銀行主檔，其餘一律邀請制）
SEED_MODE=production docker compose -f docker-compose.fullstack.yml -p mediation-prod up -d --build
```

> ⚠️ dev 與 fullstack 的資料庫是**各自獨立**的 volume；dev 灌的測試資料不會出現在 fullstack（正常隔離，非資料遺失）。

### 方式三：Oracle Cloud（OCI）等雲主機 —— 白話說明

> **為什麼特別講這段？** 因為 OCI 主機通常**只有一組公網 IP、沒有網域名稱**，而這會直接影響「網址能不能用 https」。

**一、先懂一件事：想要 `https://`（有鎖頭、加密），網站得先有一張「憑證」。**
我們用的 Caddy 會自動去免費機構 **Let's Encrypt** 申請憑證，但它有一條硬規定：

> **只發憑證給「網域名稱」（如 `abc.com.tw`），不發給「純 IP」（如 `140.238.x.x`）。**

所以 OCI 只有 IP、沒網域時，硬要走 https 會**申請不到憑證而連不上**。（這就是先前遇到的「OCI 無法連線 https」）

**二、因此本專案的預設改為「先用 http 就好」（2026-07-22 commit `https_to_http`）。**
一個環境變數 `DOMAIN` 決定走 http 還是 https：

| `DOMAIN` | 結果 | 瀏覽器打的網址 | 適用 |
|---|---|---|---|
| **不填**（預設） | 綁 `:80`、純 **HTTP**，接受任何 IP／Host | `http://<你的公網IP>` | **現在的 OCI（只有 IP）** |
| **填網域** | 自動申請憑證、走 **HTTPS** | `https://你的網域` | 將來買了網域再啟用 |

**三、在 OCI 上實際要做的三件事：**
1. 部署時**不要設 `DOMAIN`**（維持預設走 :80 HTTP）。
2. 到 OCI 後台把安全群組／防火牆的 **80 埠**對外放行。
3. 瀏覽器開 **`http://<你的公網IP>`** 即可使用。

> 補充：前端是以**同源** `/api` 呼叫後端（見第六節），所以純 IP／HTTP 情境不會有跨來源（CORS）問題。
> http 沒有加密，適合**試辦／內部驗證**；日後有網域時，只要填 `DOMAIN` 就能一鍵升級 https，無需改任何程式碼。

## 四、後端 API 一覽（含 `/api` 前綴）

> 授權欄位：「登入」＝需有效 JWT；「ADMIN」等＝另需該角色；「主辦／CO_BANK」＝依該案身分動態判定。

### 認證 `auth.ts`
| 方法 | 路徑 | 用途 | 授權 | 限流 |
|---|---|---|---|---|
| POST | `/api/auth/login` | 三層驗證登入，回傳 JWT | 公開 | 10 次 / 5 分 |
| GET | `/api/auth/me` | 取得目前登入者資料 | 登入 | — |

### 案件 `cases.ts`（整檔需登入）
| 方法 | 路徑 | 用途 | 授權 |
|---|---|---|---|
| GET | `/api/cases` | 依角色列出可見案件 | 登入（ADMIN/AUDITOR 全部；銀行僅自身相關） |
| POST | `/api/cases` | 建立案件（建立者所屬行＝主辦） | `BANK_STAFF`（非 PLATFORM） |
| GET | `/api/cases/:caseId` | 案件詳情（套用可視範圍） | 主辦／參與行／ADMIN／AUDITOR |
| POST | `/api/cases/:caseId/participants` | 邀請其他債權行（CO_BANK） | 該案主辦，且 `DRAFT` |
| PUT | `/api/cases/:caseId/declarations/:bankCode` | 代填某債權行明細 | 該案主辦（`DRAFT` 自由編輯；發布後僅能修正 DISPUTED） |
| POST | `/api/cases/:caseId/publish` | 發布並通知各行確認 | 該案主辦，`DRAFT`，須至少 1 家 CO_BANK |

### 案件流程 `caseFlow.ts`（整檔需登入）
| 方法 | 路徑 | 用途 | 授權 |
|---|---|---|---|
| POST | `/api/cases/:caseId/confirm` | 確認收到且無誤（全數確認→轉 `IN_REPAYMENT`） | 該案 CO_BANK |
| POST | `/api/cases/batch-confirm` | 批次確認多案（`caseIds[]`） | CO_BANK |
| POST | `/api/cases/:caseId/dispute` | 回報異議（`reason`） | 該案 CO_BANK |
| POST | `/api/cases/:caseId/repayments` | 登記本期各行還款（`period=YYYY-MM`, `records[]`） | 該案主辦，`IN_REPAYMENT` |
| GET | `/api/cases/:caseId/repayments` | 還款紀錄查詢 | 主辦/ADMIN/AUDITOR 全部；CO_BANK 僅本行 |
| POST | `/api/cases/:caseId/settle` | 結清 | 該案主辦，`IN_REPAYMENT` |
| POST | `/api/cases/:caseId/terminate` | 毀諾／終止（`reason`） | 該案主辦，`IN_REPAYMENT` |

### 使用者管理 `usersAdmin.ts`(整檔 ADMIN)
`GET /api/users`（可 `?status=` 篩選）、`GET /api/users/:userId/approval-logs`、`POST .../approve`、`.../reject`、`.../suspend`、`.../unlock`（解除登入鎖定）、`.../reactivate`。

### 邀請 `invitations.ts`
| 方法 | 路徑 | 用途 | 授權 |
|---|---|---|---|
| POST | `/api/invitations` | 建立邀請，回傳一次性碼與 `/register?code=` 連結 | ADMIN |
| GET | `/api/invitations` | 邀請列表（過期顯示 EXPIRED） | ADMIN |
| POST | `/api/invitations/:id/revoke` | 撤銷（僅 PENDING） | ADMIN |
| POST | `/api/invitations/validate` | 驗證邀請碼、回傳預填資訊 | 公開，20 次 / 10 分 |
| POST | `/api/invitations/accept` | 憑 NEW_ACCOUNT 碼完成註冊（密碼 ≥ 8 碼） | 公開，20 次 / 10 分 |
| POST | `/api/invitations/reset-password` | 憑 PASSWORD_RESET 碼設定新密碼（≥ 8 碼） | 公開，20 次 / 10 分 |

> 邀請碼格式 `invitationId.secret`；secret 為 18 bytes base64url，僅建立當下回傳一次，資料庫僅存 bcrypt 雜湊；效期 3 天。

- `banks.ts`：`GET /api/banks` — 銀行／機構清單。需登入。
- `notifications.ts`（需登入，僅本人）：`GET /api/notifications`（最多 100 筆＋unreadCount）、`POST /api/notifications/:id/read`、`POST /api/notifications/read-all`。

### 反向代理與 HTTP／HTTPS（Caddy，`Caddyfile`）—— 技術對照

> 白話版與 OCI 部署步驟見第三節「方式三」；此處為設定對照。

Caddyfile 於 **2026-07-22（commit `https_to_http`）** 由 `{$DOMAIN:localhost}` 改為 `{$DOMAIN::80}`：

| `DOMAIN` 設定 | Caddy 行為 | 存取方式 | 適用情境 |
|---|---|---|---|
| **未設（預設 `:80`）** | 綁 `:80`、接受任何 Host（含公網 IP），純 HTTP | `http://localhost` 或 `http://<公網IP>` | 本機驗證、**OCI 等 IP-only 雲主機**（需放行 80 埠） |
| **`DOMAIN=your.domain`** | 綁該網域並自動申請 Let's Encrypt 憑證，HTTPS | `https://your.domain` | 有正式網域的上線部署（DNS A 記錄指向主機，放行 80／443） |

- `docker-compose.fullstack.yml`：`DOMAIN` 預設 `:80`、`CORS_ORIGIN` 由 `PUBLIC_ORIGIN` 帶入。
- 前端 build 時帶 `VITE_API_BASE=""` → 以**同源** `/api` 呼叫後端，由 Caddy 代理，故純 IP／HTTP 不會有 CORS 問題。
- 兩份部署範本已依情境分檔並於檔內加註解：**OCI／IP-only（HTTP）用 `.env.production.oci.example`**、**有網域（HTTPS）用 `.env.production.domain.example`**；擇一複製為 `.env` 後填值即可，不需再手動判斷 `DOMAIN` 該不該設。
