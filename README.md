# 集中式金融資料交換平台

> 跨銀行「前置調解債權申報與還款管理」的協作平台 —— 讓多家債權銀行在同一件債務調解案中，安全地交換債權資料、確認金額、並長期追蹤還款進度。

本專案為 **2026 富邦債管聯合試辦**的技術試辦系統，屬概念驗證／試辦（POC）性質。

---

## 這個平台在解決什麼問題？

當一位債務人同時積欠多家銀行債務、進入「前置調解」程序時，實務上面臨幾個痛點：

- **資料分散**：各銀行各自持有債權明細，缺乏一致、可信的彙整管道。
- **反覆對帳**：金額須在銀行之間來回確認，容易出錯、耗時。
- **後續追蹤困難**：調解成立後的還款計畫，需要長期、逐月地跨行更新與對帳。

本平台以「**由最大債權行（主辦）主導、其他債權行確認**」的模式，把上述流程收斂到單一系統中，並保留完整的稽核軌跡。

---

## 核心概念與流程

平台圍繞「**案件（Case）**」運作，一件案件即一位債務人的一次調解：

```
建立案件 → 代填各行債權 → 邀請其他債權行 → 發布
   → 各行確認 / 回報異議 → 全數確認後進入「還款中」
   → 逐月登記還款 → 結清 或 毀諾終止
```

- **最大債權行（主辦）**：建立案件、代填各債權行的債權明細、發布案件、登記每月還款。
- **其他債權行**：收到案件後**確認**金額無誤，或**回報異議**待主辦修正。
- **平台管理／稽核**：管理帳號、檢視全案、留存操作稽核。

---

## 主要特色

- 🏦 **多銀行協作**：以主辦／其他債權行的角色分工，貼合前置調解實務。
- 🔐 **後端強制權限（Server-side RBAC）**：可見範圍與可執行動作一律由後端把關，不倚賴前端。
- ✉️ **管理員邀請制帳號**：無公開自助註冊；由管理員發一次性邀請碼建立帳號，降低冒名風險。
- 📉 **長期還款追蹤**：案件不是一次結案，而是逐月登記還款、追蹤未償餘額，直到結清或終止。
- 🧾 **完整稽核軌跡**：登入、案件異動、確認／異議、還款、帳號審核等關鍵事件皆留存。
- 🛡️ **金額精度**：金額以 `Decimal(21,4)` 儲存，避免浮點誤差。

## 技術架構概觀

| 層 | 技術 |
|---|---|
| 前端 | React 18 + TypeScript + Vite 5 + Tailwind CSS 3 + React Router 6 + Zustand |
| 後端 | Fastify 4 + Prisma 5 + Zod（JWT、bcrypt、rate-limit） |
| 資料庫 | PostgreSQL 16 |
| 部署 | Docker（db + api + web + Caddy 反向代理） |

```
├── src/                 前端（React）
├── backend/             後端（Fastify + Prisma）
│   └── prisma/          資料庫 schema、migrations、seed
├── docs/                系統文件
├── docker-compose.fullstack.yml   全棧一鍵部署
└── Caddyfile            反向代理設定
```
